# 📦 Сканер штрих-кодов

Десктопное приложение на Python для сканирования штрих-кодов через COM-порт с отображением информации о продуктах, изображений и ведением истории сканирований.

---

## 🖥️ Возможности

- Чтение данных со сканера через COM-порт в реальном времени
- Отображение названия и изображения продукта по штрих-коду
- Счётчик уникальных сканирований с визуальной индикацией
- Определение дублирующихся сканирований (подсветка синим)
- История сканирований в интерфейсе приложения
- Логирование в JSON-файл
- Настройка через внешний файл `settings.json`

---

## ⚙️ Настройка (`settings.json`)

```json
{
  "com_port": {
    "port": "COM3",
    "baudrate": 9600,
    "timeout": 1
  },
  "products": {
    "12345678901234": {
      "name": "Название продукта",
      "image": "images/product.jpg"
    }
  },
  "logging": {
    "enabled": true,
    "file": "logs/scan_log.jsonl"
  }
}
```

### Параметры

| Параметр | Описание |
|---|---|
| `com_port.port` | Имя COM-порта (например, `COM3` или `/dev/ttyUSB0`) |
| `com_port.baudrate` | Скорость передачи данных |
| `com_port.timeout` | Таймаут чтения в секундах |
| `products` | Словарь продуктов, где ключ — GTIN (символы 3–16 из штрих-кода) |
| `products.<gtin>.name` | Название продукта |
| `products.<gtin>.image` | Путь к изображению продукта |
| `logging.enabled` | Включить/выключить запись лога в файл |
| `logging.file` | Путь к файлу лога (формат JSONL) |

## 🔍 Формат штрих-кода

Приложение ожидает строку из COM-порта, из которой извлекает **GTIN** — символы с 3-го по 16-й (индексы `[2:16]`). Это стандартный формат для кодов DataMatrix и GS1-128, используемых в системах маркировки.

---

## 📊 Формат лог-файла

Каждая строка файла лога — отдельный JSON-объект:

```json
{"timestamp": "2024-01-15T10:30:00.123456", "barcode": "010460791791219321...", "product_name": "Название продукта", "is_duplicate": false, "scan_count": 5}
```

---

## 🎨 Индикация в интерфейсе

| Цвет счётчика | Значение |
|---|---|
| 🟢 Зелёный | Новый уникальный код |
| 🟡 Жёлтый | Новый уникальный код (чередование) |
| 🔵 Синий | Дубликат (повторное сканирование) |

---

## ⚠️ Возможные ошибки

**Не удалось открыть COM-порт** — убедитесь, что:
- Сканер подключён и определяется системой
- Указан верный порт в `settings.json`
- Порт не занят другим приложением

**Файл `settings.json` не найден** — создайте файл конфигурации в той же директории, что и `main.py`.

**Изображение не найдено** — проверьте путь к файлу в поле `image` в `settings.json`.
